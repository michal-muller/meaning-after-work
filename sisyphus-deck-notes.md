# Sisyphus Deck v0.1 — Scoring Logic

Sisyphus Deck is a reflective experience, not a psychological test or diagnostic instrument.

## Dimensions

The prototype tracks four reflective dimensions:

- Meaning: whether the player preserves a sense of purpose, authorship and existential significance.
- Agency: whether the player remains capable of judgment, choice and self-direction.
- Responsibility: whether the player keeps ethical and human accountability in view.
- Co-being: whether the player preserves relational presence, shared burden and human connection.

## Starting scores

Each dimension starts at 50:

- Meaning: 50
- Agency: 50
- Responsibility: 50
- Co-being: 50

Scores are bounded between 0 and 100.

## Choice effects

Each choice modifies the four dimensions.

The current scoring is interpretive and conceptual, not empirical. It is designed to make the reflective logic visible, not to measure a person.

Generally:

- Silent adaptation to speed, automation or pressure lowers Meaning, Agency and/or Co-being.
- Total refusal may preserve Agency or Meaning but can lower Responsibility or Co-being if it avoids shared negotiation.
- Human checkpoints increase Responsibility and Agency.
- Shared reflection and dialogue increase Co-being.
- Redefining the burden increases Meaning, Agency and Responsibility.
- Sharing the burden increases Co-being and Meaning.

## Result profiles

The current profile logic is:

1. Responsible Rebel  
   If Responsibility >= 68 and Agency >= 62.

2. Co-being Seeker  
   If Co-being >= 68.

3. Quiet Resister  
   If Meaning >= 64 or Agency >= 64.

4. Efficient Adapter  
   Default profile if none of the above thresholds are met.

## Important limitation

This is not a validated assessment. It should not be presented as a personality test, mental health tool or diagnostic instrument.

Preferred language:

- reflective path
- tendency
- profile
- question
- invitation

Avoid:

- diagnosis
- score
- type
- mental health assessment
- personality assessment

## Data ethics

The current version stores nothing, sends nothing and uses no database. All choices remain in the visitor's browser during the session only.
